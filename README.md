# תרגול הנדסת תוכנה — Software-Engineering Quiz

משחק תרגול (עברית, RTL) לקורס *הנדסת תוכנה* (63301). שאלה אמריקאית עם אפשרויות,
בוחרים תשובה ומקבלים משוב/ציון. אתר **סטטי לחלוטין** (HTML+CSS+JS) — נפתח מקומית או
מתפרסם ב-GitHub Pages ללא שום שלב בנייה. בנוי במתכונת אפליקציית *מבוא למדעי הנתונים*.

## הפעלה מקומית
פותחים את `index.html` בדפדפן (דאבל-קליק). הנתונים נטענים מ-`questions.js` כמשתנה
גלובלי, ולכן זה עובד גם מ-`file://` (ללא צורך בשרת).

## פרסום ל-GitHub Pages
1. צרו ריפו והעלו את **תוכן התיקייה** (לפחות `index.html`, `styles.css`, `app.js`, `questions.js`; ו-`learn.js` + `images/` למצב הלימוד).
2. ב-GitHub: **Settings → Pages → Deploy from a branch**, ענף `main`, תיקייה `/ (root)`.
3. `.nojekyll` מונע עיבוד Jekyll. כל הנתיבים יחסיים; אין תלות רשת/CDN — עובד גם אופליין.

## מאפיינים
- **שני מצבי תרגול:** *תרגול חופשי* (משוב מיידי + הסבר) ו-*מצב מבחן* (זמן קצוב, ציון ופירוט).
- **סינון לפי נושא** (11 נושאים: מבוא, Agile/SCRUM, דרישות, UML, ארכיטקטורה, תבניות עיצוב, SOLID, בדיקות/TDD, OOP/Java, מטריקות, אספקה).
- **מבחן ספציפי מלא** — לשחזר מבחן עבר שלם, או תרגול אקראי מכל המאגר.
- **ערבוב סדר התשובות** בכל הצגה (Fisher–Yates) — מנטרל את "התשובה תמיד ראשונה" של מבחני טופס-אפס.
- **ערבוב בזמן בנייה** — גם ה-`correctIndex` השמור מעורבב, כך שגם קריאה של `questions.json` לא חושפת "תמיד ראשון".
- **הסרת כפילויות** — שאלות שחוזרות בין מבחנים מוצגות פעם אחת בתרגול לפי נושא, אך מבחן מלא נשאר שלם.
- **מצב לימוד 📖** — סיכום מסודר לפי נושאי הקורס (מ-`learn.js`).
- **מצב בהיר/כהה** (נשמר ב-`localStorage`, ברירת מחדל לפי המערכת) ו-**מעקב התקדמות**.
- תמיכה במקלדת: ספרות `1–8` לבחירה, `Enter`/`←` למעבר.

## מקור הנתונים ואיכותם
`questions.js` נבנה אוטומטית מארכיון מבחני הקורס (2019–2025). התהליך:
1. כל מבחן עובר **רינדור לתמונות עמוד** (RTL עברית יוצא משובש כטקסט — התמונות הן המקור המדויק).
2. סוכן־מבנה קורא את תמונות העמודים וממיר לשאלות JSON — **ללא המצאה**; שאלות שלא ניתן לאמת מסומנות.
3. **קביעת התשובה הנכונה** לפי אחד משלושה: מבחן **טופס-אפס** (תשובה = אפשרות ראשונה) · **קובץ פתרונות** (התשובה המסומנת) · אחרת **נגזרת** מידע הנדסת התוכנה ומסומנת "לא רשמי".
- שאלות המסומנות **"מחוון רשמי"** — התשובה מקובץ פתרונות/טופס-אפס.
- שאלות **"תשובה לא רשמית"** — נגזרו; מומלץ לאמת מול המקור.
- שאלות התלויות בתרשים שלא ניתן לתמלול הוצאו אוטומטית עד לצירוף תמונה.

פירוט מלא (כמות לפי נושא ולפי מבחן, ורשימת שאלות מסומנות): `tools/build_report.md`.

## בנייה מחדש של המאגר (אופציונלי)
```
python tools/extract_exams.py        # PDF/DOCX -> תמונות עמוד + טקסט גולמי (tools/raw/)
# מבני השאלות (tools/raw/<CODE>.json) הופקו ע"י סוכני-מבנה מתמונות העמודים
python tools/validate.py             # בדיקות תקינות של קובצי ה-raw
python tools/build_questions.py      # מיזוג -> questions.js / questions.json / build_report.md
python tools/build_learn.py          # se_summary.md -> learn.js (מצב הלימוד)
node   tools/smoke.js                # בדיקת נתונים + ערבוב-בנייה + אינוריאנט ניקוד
node   tools/verify_ui.js            # בדיקת UI בדפדפן (puppeteer-core)
node   tools/verify_theme.js         # בדיקת מצב בהיר/כהה
node   tools/verify_learn.js         # בדיקת מצב הלימוד
```
> להרצת בדיקות הדפדפן דרוש Chrome מותקן. אפשר להצביע על נתיב אחר עם `CHROME_PATH`.

## מבנה
```
index.html · styles.css · app.js · questions.js   ← האתר עצמו (זה מה שמפרסמים)
learn.js · images/                                 ← מצב הלימוד
questions.json                                     ← אותם נתונים, לשימוש חוזר
docs/                                              ← build_plan.md · exam_inventory.md
tools/                                             ← צינור עיבוד הנתונים
```

> ⚠️ הופק לצורכי לימוד אישי. תשובות "לא רשמיות" נגזרו אוטומטית — יש לאמת מול חומר הקורס.
